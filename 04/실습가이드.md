# 04. QoS / 지연 인식 스케줄링

## 목표
단순 PF에 **대기시간(HOL delay)** 또는 **QoS 클래스 가중치**를 넣어  
급한 트래픽이 먼저 나가게 만든다.

## 구성
| UE | qos_class | 의미 |
|----|-----------|------|
| 0 | GBR (가중 큼) | 지연에 민감 |
| 1 | Non-GBR | best effort |
| 2 | GBR | 지연에 민감 |

매 TTI, 선택되지 않은 UE의 `hol_delay_ms`가 1씩 증가한다.

## metric (간단 M-LWDF 유사)
```
metric = qos_weight * hol_delay_ms * (inst / (avg_rate + bias))
```

## 실습 과제
1. `qos_weight`를 바꿔 GBR 우선이 보이는지 확인
2. delay를 metric에서 빼면(=순수 PF) Non-GBR가 얼마나 끼어드는지 비교
3. (선택) deadline 초과 카운터를 추가해 본다
