# 06. Strict Priority + GBR floor

## 목표
GBR 미달 UE를 **Strict Priority**로 먼저 서비스하고, 나머지 자원은 **PF**로 Non-GBR·GBR(이미 floor 충족)에 할당한다.

## 후보 알고리즘
Strict Priority, GBR-aware

## 구성
| UE | qos | gbr_floor |
|----|-----|-----------|
| 0 | GBR | 4500 bytes |
| 1 | Non-GBR | — |
| 2 | GBR | 4500 bytes |

## 비교
- `GBR_strict` : floor 미달 GBR 우선 → 잔여 PF
- `PF_only` : 04/05와 동일 PF만

## 실습 과제
1. `gbr_floor`를 키워 GBR_met=1 달성 TTI 관찰
2. `PF_only`와 UE1 picks·GBR_met 차이 비교
