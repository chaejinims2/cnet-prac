# 02. RR / Max-CQI / PF 비교

## 목표
동일 UE·동일 CQI 시퀀스에서 Round Robin, Max-CQI, PF를 **한 번에** 돌리고  
총 throughput(처리량)과 Jain fairness(공평성)를 비교한다.

## Jain index
```
J = (sum r_i)^2 / (N * sum r_i^2)
```
1에 가까울수록 공평, 1/N에 가까울수록 불평등.

## 실습 과제
1. `bash build_run.sh` 한 번 실행 → RR / Max-CQI / PF 결과가 순서대로 출력됨
2. 각 알고리즘 `summary`의 total_bytes, Jain 값을 메모·비교
3. Max-CQI에서 어느 UE가 독점하는지 관찰
4. (선택) `RNG_SEED`를 바꾸면 fairness가 어떻게 변하는지 확인

## 후기
이전에는 pf 알고리즘 기반 스케줄러를 구현했었고 이번에는 거기에 더해 MAX-CQI 알고리즘과 RR 기반 3개의 스케줄러 를 구현했다. 세 알고리즘 모두 공통 시드로 구현했었고, 세 알고리즘을 비교하니 RR 알고리즘이 가장 공평하고, MAX-CQI 알고리즘이 가장 빠르고, PF 알고리즘이 가장 효율적이었다. 하지만 UE가 3개뿐이라 실제 더 많은 UE가 있을 때 어떤 알고리즘이 가장 좋을지 확인해보고 싶다.
