# 05. 트래픽 도착 모델 + 스케줄링

## 목표
초기 buffer만 고갈시키던 01과 달리, **매 TTI 데이터가 유입**되는 모델을 붙인다.

## 도착 모델 (간단)
```
arrival = ARRIVAL_MEAN + (rand() % ARRIVAL_JITTER) - ARRIVAL_JITTER/2
```
UE마다 mean이 다르면 “무거운 사용자 / 가벼운 사용자” 실험이 된다.

## 흐름 (1 TTI)
1. `arrive_traffic()` — 버퍼 증가
2. CQI fade
3. PF(또는 02/03 알고리즘)로 스케줄
4. 통계: queue length, drop(버퍼 상한 초과)

## 실습 과제
1. `ARRIVAL_MEAN_*`를 키워 기지국이 얼마나 backlog를 쌓는지 관찰
2. `BUFFER_CAP`를 낮춰 drop이 나는 지점 찾기
3. (선택) 02의 RR/Max-CQI를 붙여 load 높을 때 누가 무너지는지 비교
